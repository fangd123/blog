Python Scrapy框架Ubuntu 15.04下安装小记

2015-09-28
生活
约 755 字 预计阅读 2 分钟 次阅读

文章目录

我这人有个坏习惯，总是喜欢装最新的东西，殊不知最新的不一定是最稳定的，有时候会出现一些Error和Bug，另外，针对与桌面系统的Ubuntu和针对与服务器的CentOS比起来，组件的齐全度上确实还是差了一些，非常多的组件需要自己安装。想我之前用CenOS7的时候直接用pip install scrapy就把Scrapy安装好了，但是在Ubuntu下边，光是安装各种包就花费了不少的时间。不过没办法，谁让Ubuntu是最受欢迎的Linux操作系统呢。

安装过程我这里就不再赘述了，一般是按照官网推荐的使用pip install安装，Ubuntu软件库中自带的Scrapy因为版本比较老了没有更新，官方不建议安装，如果确实享用apt-get install安装的话，官网也有相对应的教程，照着做就是了。

在安装的过程中，会出现很多很多很多（没错，非常多）错误，究其原因，Ubuntu许多开发相关的库都没有安装，需要我们手动进行。不过还好，错误提示里边一般都会写明出现了什么错误，将错误在谷歌中搜索即可。

  • 有时候是找不到.h头文件，这个时候需要谷歌对应的头文件，寻找对应的安装包安装。

  • 也有这样的错误

UnicodeDecodeError: ‘ascii’ codec can’t decode byte 0xc3 in position 0: ordinal not in range(128)

这个就需要对于python的配置文件进行修改，具体修改方法如下：

找到Python安装目录\Lib下的 mimetypes.py 文件，找到大概256行（你可以用Notepad++的搜索功能）的default_encoding = sys.getdefaultencoding()在前边增加如下代码：

1
2
3
4

if sys.getdefaultencoding() != 'gbk':
	reload(sys)
	sys.setdefaultencoding('gbk')
default_encoding = sys.getdefaultencoding()

感谢hugleecool的博客提供解决方案

在经过一番折腾之后，最后总算是顺利安装上去了，没有报错。说实话，配置都这么花费时间我觉得真心应该像Laravel学习直接给一个虚拟机镜像，不把时间花费在配置上，集中精力进行开发。
